Cognitive Computing and AI Concepts

Question 1: What are the pros and cons of cognitive computing?

Pros:

  • Can handle complex tasks that traditionally required human intelligence.
  • Ability to process and learn from large amounts of data.
  • Operates 24/7 without fatigue.
  • Augments human capabilities in decision-making.

Cons:

  • Tasks easy for humans can be very difficult for cognitive systems ("Moravec's Paradox").
  • Challenges in true understanding versus pattern matching.
  • May be subject to algorithmic bias.
  • May have issues with goal/reward alignment.

Question 2: Do you know about any real-world applications of AI?

  • Vision and Image Processing: OCR, face detection, vehicle safety systems, visual search, image generation.
  • Natural Language Processing: Text-to-speech, machine translation, ChatGPT, etc.
  • Robotics: Mars rovers, autonomous vehicles, robot pets, personal assistants.
  • Smart Systems: Intelligent thermostats, spam filters, automated taxi systems.

Question 3: Let us examine the rationality of various vacuum-cleaner agent functions.

A rational agent selects actions that maximize its expected performance measure. For the vacuum-cleaner agent:

  1. Simple Vacuum-Cleaner Rationality:
    2. - Perceives location and dirt status (e.g., [A, Dirty]).
    - Actions available: Left, Right, Suck, NoOp.
    - Rational if it chooses Suck when there's dirt and moves to explore other squares when clean.
  2. Movement Costs One Point:
    3. - Maintains internal state (model-based reflex agent).
    - Balances movement cost against cleaning benefit.
    - Performance measure accounts for cleanliness and movement costs.
  3. Dynamic Environments:
    4. - Learning becomes crucial.
    - The agent should learn: * Dirt accumulation patterns. * Environment geography. * Optimal cleaning strategies.

Question 4: Define the following terms...

  1. Agent: An entity that perceives its environment through sensors and acts upon it through actuators.
  2. Agent Function: Maps percept sequences to actions (`f: P* → A`).
  3. Agent Program: The concrete implementation of the agent function that runs on physical architecture.
  4. Rationality: A rational agent selects actions maximizing expected performance based on percepts and knowledge.
  5. Autonomy: The ability to operate independently without human intervention.
  6. Reflex Agent: Uses simple condition-action rules without internal state or memory.
  7. Model-Based Agent: Maintains internal state to track aspects of the environment using a transition model.
  8. Goal-Based Agent: Plans actions to achieve explicit goals.
  9. Utility-Based Agent: Assigns utility values to states and maximizes expected utility.
  10. Learning Agent: Improves performance over time through learning mechanisms (critic, learning element, problem generator).

Question 5: Is a thermostat an instance of a simple reflex agent, a model-based reflex agent, or a goal-based agent?

The thermostat is an instance of a simple reflex agent, as it operates on fixed condition-action rules:

  1. Rules:
    2. - `IF (temperature ≤ setting - 3) THEN turn_on_furnace`.
    - `IF (temperature ≥ setting + 3) THEN turn_off_furnace`.
  2. Characteristics of Simple Reflex Agents:
    3. - No internal state maintenance required.
    - Direct mapping from current percept to action.
  3. Contrasts with Other Agents:
    4. - Unlike model-based agents: No state tracking or prediction.
    - Unlike goal-based agents: No reasoning about future states.
    - Unlike utility-based agents: No evaluation of outcomes.

Question 6: Your goal is to navigate a robot out of a maze...

Using intelligent agent concepts:

  1. Initial Problem Formulation:
    2. - State space includes robot's location `(x,y)`, orientation (North/East/South/West), actions (Turn/Move Forward), and percepts (Wall detection).
    - Performance measure: Steps to exit.
  2. Intersection-Based Reformulation:
    3. - Reduced state space by tracking only intersection points and orientation.
  3. Direction-Based Reformulation:
    4. - Further reduced state space by tracking only intersection points without orientation.
  4. Real-World Simplifications:
    5. Assumptions based on environment properties include fully observable, deterministic movements, discrete space, static environment, and no physical constraints like momentum or friction.
